# Release Trust

This document explains how users should identify official OpenAssetWatch releases.

## Official source

The official source repository is:

https://github.com/PKXLIVE/openassetwatch

## Official release expectations

Official OpenAssetWatch releases should include, where practical:

- GitHub Release notes
- Source tag
- Build artifacts
- SHA-256 checksums
- Manifest metadata
- SBOM artifact
- Artifact attestations for release workflows that support them

## Unofficial builds

Forks and modified builds must clearly identify themselves as unofficial or modified and must not claim to be official OpenAssetWatch releases.

## Reporting suspicious repositories or packages

Report suspected impersonation, malicious clones, or confusing package names through GitHub Issues or private vulnerability reporting when security-sensitive.
