# Contributing to OpenAssetWatch

Thank you for helping improve OpenAssetWatch.

## Contribution license

By submitting a pull request, issue, suggestion, patch, or other contribution to this repository, you agree that your contribution is provided under the Apache License, Version 2.0, unless you explicitly state otherwise in writing.

## Development workflow

1. Fork the repository.
2. Create a feature branch.
3. Keep changes focused and reviewable.
4. Add or update tests where practical.
5. Open a pull request against `main`.

## Required checks

Pull requests should pass the relevant CI checks before merge.

## Security-sensitive changes

Changes touching collectors, installers, release workflows, authentication, permissions, network behavior, or package signing may require extra maintainer review.

## No secrets

Do not commit credentials, API keys, tokens, private keys, certificates, customer data, or private infrastructure details.

## Code of conduct

All contributors are expected to follow `CODE_OF_CONDUCT.md`.
